O que é um SGBD? Tipos e Aplicações?

Um Sistema de Gerenciamento de Banco de Dados (SGBD) é uma plataforma software que possibilita a criação, manutenção e controle de bancos de dados de forma organizada e segura. Atua como intermediário entre os usuários e os dados armazenados, garantindo integridade, confiabilidade e eficiência no acesso às informações.

Os SGBDs classificam-se principalmente em relacionais (SQL) e não-relacionais (NoSQL), cada um com características distintas. Os sistemas relacionais, como MySQL e PostgreSQL, organizam os dados em tabelas com esquemas fixos e relações definidas, sendo ideais para aplicações que exigem transações complexas e consistência rigorosa.

Já os sistemas não-relacionais, como MongoDB e Redis, oferecem modelos flexíveis de armazenamento, adequados para cenários que demandam escalabilidade horizontal e lidam com dados variáveis ou não estruturados. As aplicações abrangem desde sistemas corporativos e comércio eletrônico até análise de dados em tempo real e soluções de Internet das Coisas, sendo componentes fundamentais em praticamente todos os ambientes que necessitam de armazenamento e recuperação eficiente de informações.

Quais os principais bancos de dados?



Dentre as soluções mais relevantes no mercado, destacam-se os bancos relacionais MySQL, amplamente adotado em aplicações web pela sua simplicidade e custo acessível; PostgreSQL, reconhecido por recursos avançados e conformidade com padrões SQL; Oracle Database, voltado para ambientes corporativos de grande porte; e Microsoft SQL Server, integrado ao ecossistema de tecnologias Microsoft. Na esfera não-relacional, MongoDB sobressai pelo modelo de documentos flexíveis; Redis pelo armazenamento em memória para alta performance; Cassandra pela capacidade de distribuição em larga escala; e Elasticsearch por funcionalidades especializadas em busca e análise. A seleção entre estas opções depende de requisitos como volume de dados, estrutura informacional, necessidades de desempenho e contexto operacional específico.

O que é um dicionário de dados? Quais os tipos de dados de um banco de dados?

Um dicionário de dados consiste em um repositório de metadados que descreve a estrutura, organização e regras associadas aos dados armazenados. Funciona como documentação de referência para administradores, desenvolvedores e analistas, contendo detalhes sobre tabelas, colunas, tipos de dados, relacionamentos, restrições e permissões. No que concerne aos tipos de dados, os bancos suportam categorias fundamentais para assegurar armazenamento apropriado e operações eficientes. Incluem-se tipos numéricos como INT para valores inteiros, DECIMAL para precisão decimal exata e FLOAT para aproximações de ponto flutuante; tipos de texto como VARCHAR para cadeias de caracteres variáveis e TEXT para conteúdos extensos; tipos temporais como DATE e TIMESTAMP para datas e horários; tipos lógicos como BOOLEAN para valores verdadeiro/falso; e tipos binários como BLOB para armazenamento de objetos. A correta especificação desses tipos é crucial para a integridade e otimização do banco de dados.

O que é uma modelagem de banco de dados? E o modelo DER? MER?

A modelagem de banco de dados refere-se ao processo de projetar a estrutura lógica e física de um banco, definindo como os dados serão organizados, relacionados e acessados. Trata-se de etapa fundamental no desenvolvimento de sistemas, assegurando que o banco atenda aos requisitos de negócio com eficácia, minimizando redundâncias e inconsistências. O modelo DER (Diagrama Entidade-Relacionamento) é uma ferramenta visual que representa as entidades (objetos do domínio, como "Cliente" ou "Produto"), seus atributos (propriedades das entidades, como "nome" ou "código") e os relacionamentos entre elas (como "pertence a" ou "é vinculado a"). O modelo MER (Modelo Entidade-Relacionamento) amplia esse conceito, incorporando detalhes adicionais como cardinalidades (um-para-um, um-para-muitos) e generalizações, servindo como base para a derivação do modelo lógico que será subsequentemente implementado no SGBD escolhido.
